初识查询

上一篇文档中，我们定义的Repository接口并没有定义任何方法，因为从基接口继承到的行为足够我们展示本文的示例了。

查询简单对象​

在任何位置注入bookStoreRepository，如此调用

  • Java
  • Kotlin
List<BookStore> stores = bookStoreRepository.findAll();
System.out.println(stores);

查询复杂数据结构​

在任何位置注入bookStoreRepository，如此调用

  • Java
  • Kotlin
List<BookStore> stores = bookStoreRepository.findAll(
    BookStoreFetcher.$
        .allScalarFields()
        .books(
            BookFetcher.$
                .allScalarFields()
                .authors(
                    AuthorFetcher.$
                        .allScalarFields()
                )
        )
);
System.out.println(stores);
val stores = bookStoreRepository.findAll(
    newFetcher(BookStore::class).by {
        allScalarFields()
        books {
            allScalarFields()
            authors {
                allScalarFields()
            }
        }
    }
)
println(stores)

必要说明：

  • findAll方法的参数叫对象抓取器。Jimmer可以查询任意复杂的数据结构，对象抓取器用于指定被查询数据结构的形状。

  • Java代码中，BookStoreFetcher、BookFetcher和AuthorFetcher是预编译器自动生成的代码，包名和对应实体一致。

  • Kotlin代码中

    • newFetcher为org.babyfish.jimmer.sql.kt.fetcher.newFetcher
    • by函数是扩展函数，是预编译器自动生成的代码，包名和对应实体一致。

  • allScalarFields表示盲目查询实体的所有非关联字段，不做精细控制。

可见，指定对象抓取器后，Jimmer会完成逐层查询，直到整个数据结构查询完成为止。

提示

对象抓取器优势：

  • 非关联属性也可以精细控制（注意：本例子未展示，只是用allScalarFields盲目查询所有非关联字段），这种能力对以下两种情况非常有用：

    • 实体字段非常多，但是当前业务只需要很少一部分。

    • 权限管理系统具备字段级限制，当前用户并没有权限访问所有字段。

  • 关联属性的查询是批量化的，对于每一层关联，Jimmer都是用批量查询一条SQL完成，解决N + 1查询问题

    然而，如果某个批次数据太多，Jimmer会自动截断成多个批次，不强求必须一条SQL完成。批次大小可以配置：

    • 非集合关联（一对一、多对一）：由spring boot的配置属性jimmer.default-batch-size控制，默认256
    • 集合关联（一对多、一对多）：由spring boot的配置属性jimmer.default-list-batch-size控制，默认16

    注意：Oracle下，配置值不得大于1000

  • 不仅可以精细控制每一个被映射到数据库的属性是否需要，还可以控制业务计算属性是否需要。

    所谓业务计算属性，指不能从数据库简单映射的属性，必须靠业务系统计算才能得到的复杂属性。

    业务计算属性属性将会在后续教程中讲解，这里读者明白对象抓取器有这个能力即可。

  • Jimmer可以为关联属性/计算属性启用缓存。一旦某些关联属性/计算属性启用了缓存，对象抓取器将会和自动缓存交互，而不只是简单是生成SQL语句。

    无论关联属性/计算属性的查询是单纯的SQL行为，还是需要混入缓存行为，对对象抓取器的使用者透明。

  • 如果聚合根对象是分页查询的结果（注意：本例子未展示，只是用java.util.List返回所有数据，并未使用分页查询），对象抓取器都所有行为都发生在分页后。

递归查询​

上文中，我们演示的复杂数据结构查询，但数据结构的深度是已知的。

接下来，我们演示无限深度数据结构查询，递归查询。

在任何位置注入treeNodeRepository，如此调用

  • Java
  • Kotlin
TreeNode treeNode = treeNodeRepository.findNullable(
    1L,
    TreeNodeFetcher.$
        .allScalarFields()
        .childNodes(
            TreeNodeFetcher.$
                    .allScalarFields(),
            cfg -> cfg.recursive()
        )
);
System.out.println(treeNode);
val treeNode = treeNodeRepository.findNullable(
    1L,
    newFetcher(TreeNode::class).by {
        allScalarFields()
        childNodes({
            recursive()
        }) {
            allScalarFields()
        }
    }
)
println(treeNode)
